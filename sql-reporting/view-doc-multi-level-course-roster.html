Multi-Level Course Roster and Scheduling Report

Technical documentation for a sanitized operational roster and scheduling query.

View Query View Sample Results Back to SQL Portfolio

Overview

This SQL example demonstrates how a multi-table operational report can combine enrollment, scheduling, course, term, school, and instructor data.

The query is based on a real reporting workflow but removes organization-specific names, identifiers, course codes, and personnel information.

Purpose

The report identifies actively enrolled records that meet grade-level and course-specific participation rules for a selected reporting year.

Report Fields

  • Student identifier
  • SIS identifier
  • First and last name
  • Grade level
  • School
  • Course number and name
  • Section number
  • Term
  • Instructor

Tables Used

  • students
  • cc
  • terms
  • schools
  • sections
  • courses
  • teachers

SQL Concepts Demonstrated

  • Seven-table INNER JOIN structure
  • Parameterized reporting-year filtering
  • Active-record filtering
  • Grouped AND and OR logic
  • Grade-specific business rules
  • IN and NOT IN filters
  • Duplicate prevention with DISTINCT
  • Multi-column sorting

Business Logic

The query includes records only when:

  1. The student is actively enrolled.
  2. The course belongs to the selected reporting year.
  3. The instructor is not part of the exclusion list.
  4. The grade level matches an approved set of course codes.

Grouped conditions allow each grade band to use different course-selection rules without repeating the entire query through multiple UNION statements.

Sanitization

The public version replaces:

  • School names
  • Organization identifiers
  • Course numbers
  • Instructor identifiers
  • Reporting-year values
  • Personally identifiable information

The original query structure and reporting logic were retained.

Potential Uses

  • Course rosters
  • Program participation reports
  • Employee training assignments
  • Membership eligibility reports
  • Certification tracking
  • Department scheduling
  • Access-provisioning audits
