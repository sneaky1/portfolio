Active Enrollment and Effective-Date Report

Documentation for a sanitized point-in-time enrollment and eligibility report.

View Query View Sample Results Back to SQL Portfolio

Overview

This Oracle SQL example demonstrates point-in-time reporting using entry dates, exit dates, effective dates, and a selected report date.

The public version preserves the original date-handling and reporting logic while replacing organization-specific details.

Purpose

The report identifies participants who were active on a selected reporting date.

Report Fields

  • Person number
  • First and last name
  • Eligibility category
  • Effective date
  • Entry date
  • Exit date
  • Calculated enrollment status

Table Used

  • program_participants

SQL Concepts Demonstrated

  • Oracle date comparisons
  • Point-in-time status logic
  • Null exit-date handling
  • Bind variables
  • CASE classification
  • TO_CHAR date formatting
  • Active-record filtering
  • Multi-column sorting

Business Logic

A participant is active when:

  1. The entry date is on or before the selected report date.
  2. The exit date is blank or on or after the selected report date.

The query uses the bind variable :report_date so the same report can be reused for different historical dates.

Date Filtering Versus Formatting

The query compares the original date values in the WHERE clause.

The TO_CHAR function is used only in the SELECT list to format dates for display.

This avoids converting dates to text before comparison, which can produce incorrect results and reduce performance.

Null Exit Dates

A null exit date represents an open or ongoing enrollment. 

p.exit_date IS NULL
OR p.exit_date >= :report_date

This condition includes both ongoing records and records that had not ended by the selected report date.

Sanitization

The public version replaces:

  • Student-specific terminology
  • Organization identifiers
  • Program names
  • Eligibility values
  • System-specific table and field names
  • Personally identifiable information

Potential Uses

  • Student enrollment
  • Employee assignments
  • Program participation
  • Benefit eligibility
  • Membership status
  • Subscription activity
  • Historical compliance reporting
